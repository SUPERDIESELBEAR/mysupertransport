## Goal
Confirm every gold badge button ("CTA") rendered in outgoing emails is (a) a real, absolute HTTPS link, (b) points at a route that exists in the SUPERDRIVE React app, and (c) lands the recipient on the correct screen (with the right tab / anchor / auth context). Fix any that are broken.

## Scope — buttons to check
All gold-button CTAs generated by these edge functions and shared templates:

**Auth / invites**
- `auth-email-hook` + `_shared/email-templates/*` (signup, magic-link, recovery, reauthentication, invite, email-change)
- `invite-applicant`, `invite-operator`, `invite-staff`, `invite-truck-owner`, `resend-invite`, `resend-application-link`
- `launch-superdrive-invite` (both "Open SUPERDRIVE" and "Open Dashboard" variants)

**Onboarding & operator lifecycle**
- `notify-onboarding-update` (all 10+ milestone copies — every `cta:` in the file)
- `notify-application-moved-to-pending`
- `notify-document-update`, `notify-upload-attention`, `notify-idle-operators`
- `notify-pay-setup-submitted`, `send-payroll-docs` ("View Document →" per row)
- `notify-new-message` ("Open Messages")
- `notify-pwa-install`
- `send-lease-termination`, equipment-return-instructions template
- `send-insurance-request`, `send-cert-reminder`, `cron-cert-reminders`

**PEI**
- `pei-request-initial`, `pei-request-follow-up`, `pei-request-final-notice` templates
- `pei-release-fcra` (public tokenized viewer link)

**Marketing / misc**
- `send-birthday-anniversary`, `send-staff-birthday-message`
- `send-resource-email`, `send-release-note`, `send-operator-broadcast` (custom `ctaUrl`)
- `send-application-correction-email` ("Review & approve changes")
- `send-test-email`, `preview-transactional-email`
- `download-qpassport` ("Open My Portal", "Download")

## Method
1. **Static audit** — grep every `cta:`, `href=`, and `buildAppUrl(...)` call above; build a table of `{function, CTA label, target path, expected React route}`.
2. **Route existence check** — cross-reference each target path against the router in `src/App.tsx`, `OperatorPortal.tsx` (tab query params), `ManagementPortal.tsx`, and any public routes (`/apply`, `/install`, `/pei/fcra/:token`, `/dashboard`, `/login`). Confirm each tab value (`?tab=progress|documents|ica|notifications|messages`) is a real case in the corresponding portal switch.
3. **Runtime spot-check** — for high-traffic templates (onboarding milestone, new message, PEI initial, correction email, launch invite, payroll docs), call `preview-transactional-email` / render locally and click the button in Playwright to confirm it lands on the right screen without a redirect loop or 404. Include one authenticated case (session restored) and one signed-out case (should route through login and back).
4. **Report** — produce a checklist of each CTA with PASS / FAIL / NOTE, e.g. "invite-applicant → /apply ✅", "notify-onboarding-update decal step → /operator?tab=documents but no decal anchor — scrolls to top ⚠️".
5. **Fix** — for each FAIL, correct the URL builder (usually swap the hard-coded path or add the missing anchor / tab param). Re-run the runtime check.

## Deliverable
- Audit table checked into chat.
- Code patches for any broken CTAs (edge functions only; no UI changes unless a route is genuinely missing).
- Short confirmation that every gold button opens the correct in-app destination.

## Out of scope
- Copy / wording changes to CTAs (unless the label is misleading about where it goes).
- Non-gold links (mailto:, plain-text footer links, unsubscribe) — only verified as present, not re-styled.
- Email deliverability, DNS, or template branding.
